# Railway를 이용한 홈페이지 배포 가이드 (완전 초보자용)

웹사이트를 실제 인터넷에 호스팅하기 위해 **Railway** 서비스를 사용하는 방법입니다. 
관리자 페이지에서 업로드하는 이미지와 뉴스, 부동산 매물 데이터베이스(SQLite) 정보가 날아가지 않도록 **볼륨(Volume)**을 붙여서 영구 보존하는 부분까지 상세하게 설명합니다.

---

## 📦 0단계: GitHub에 올릴 파일과 폴더 구조 이해하기

GitHub에 코드를 올리기(Push) 전, **"이게 다 통째로 올라가는 건가?"** 하고 헷갈리실 수 있습니다.
정답은 **"아닙니다!"** 

현재 프로젝트 디렉토리 최상단에는 숨김 파일인 `.gitignore`가 설정되어 있습니다. 이 파일이 **불필요하거나 보안상 중요한 파일들이 GitHub에 올라가지 않도록 자동으로 막아줍니다**.

### ❌ GitHub에 절대 올라가지 않는(제외되는) 항목들
- `node_modules/` : 수백 메가바이트가 넘는 라이브러리 설치 폴더 (Railway가 빌드할 때 알아서 새로 다운받습니다.)
- `dist/` : 빌드된 결과물 폴더
- `.env` : 관리자 비밀번호나 보안키가 적힌 파일
- `data/` : 로컬 컴퓨터에서 작성되었던 SQLite 데이터베이스 파일과 백업 파일 (`*.db`)
- `server/uploads/` : 로컬 컴퓨터에서 테스트를 위해 임시로 올렸던 사진 파일들

### 🟢 GitHub에 실제로 올라가는 필수 폴더 및 파일들
아래 파일들은 코어 소스코드이므로 반드시 올라가야 합니다. (아래 명령어 `git add .` 시 자동으로 걸러져서 아래 항목들만 올라가게 됩니다.)
- `src/` : 리액트(React) 프론트엔드 소스 파일들이 모여있는 핵심 폴더
- `server/` : 백엔드 서버(Express), DB 연결 소스가 있는 핵심 폴더 (uploads 폴더 내부 제외)
- `public/` : 파비콘(아이콘)이나 정적 이미지 등 공통 구성요소
- `package.json` & `package-lock.json` : Railway에게 '무엇을 설치하고 실행할지' 알려주는 명세서
- `tsconfig.json`, `vite.config.ts` 등 각종 설정 파일들

이 구조만 알고 계시면, 앞으로 안심하고 소스 코드 전체를 전부 추가(`git add .`) 하셔도 무방합니다!

## 🚀 1단계: 준비사항 (GitHub에 코드 올리기)

Railway는 GitHub에 있는 코드를 가져가서 자동으로 배포해 줍니다. 따라서 작업하신 프로젝트가 GitHub에 올라가 있어야 합니다.

1. **GitHub 계정 생성 및 로그인**: [github.com](https://github.com/)
2. **새로운 Repository 생성**:
   - 우측 상단의 `+` 버튼 -> `New repository` 클릭
   - Repository name에 원하는 이름 입력 (예: `realestate-homepage`)
   - `Create repository` 클릭
3. **VSCode(현재 작업중인 에디터) 터미널**에서 아래 명령어를 차례대로 입력하여 코드를 GitHub으로 전송합니다.
   ```bash
   git init
   git add .
   git commit -m "첫 번째 배포 준비 완료"
   git branch -M main
   git remote add origin https://github.com/본인아이디/본인저장소이름.git
   git push -u origin main
   ```
   *참고: 위 작업은 최초 1회만 설정하면 되며, 이후 코드를 수정하고 업데이트할 때는 `git add .`, `git commit -m "수정사항"`, `git push` 세 가지 명령어만 치시면 됩니다.*

---

## 🛠️ 2단계: Railway 시작 및 프로젝트 생성

1. **Railway 가입 및 로그인**: [railway.app](https://railway.app/)
   - `Login` 메뉴에서 `GitHub`으로 로그인하는 것을 추천합니다.
2. 메인 대시보드에서 우측 상단의 **[New Project]** 버튼을 클릭합니다.
3. 목록에서 **[Deploy from GitHub repo]**를 선택합니다.
4. 방금 전 생성한 본인의 GitHub Repository 이름을 찾아 클릭합니다.
5. 화면에 보라색 버튼 **[Deploy Now]**가 나오면 클릭합니다. (앱이 생성되고, 1차 배포가 자동으로 시작됩니다.)

## 💾 3단계: 볼륨(Volume) 설정하기 (중요)

> 💡 **왜 볼륨이 필요한가요?**
> 클라우드 서버 특성상, 배포(업데이트)될 때마다 서버가 초기화됩니다. 데이터베이스 파일(`.db`)과 업로드한 이미지(`/uploads`)를 날리지 않고 영구적으로 보존하기 위해서는 가상의 외장하드 같은 '볼륨(Volume)'을 연결해 주어야 합니다.

현재 코드는 `DATA_DIR` 변수를 참조해 `/app/data` 경로를 사용하도록 모두 업데이트되어 있습니다! 설정만 추가해주시면 됩니다.

1. Railway 대시보드에 띄워진 본인의 **Project** 화면으로 갑니다.
2. 여러 개의 상자(카드) 중 본인의 저장소 이름이 적힌 카드를 클릭하여 설정 화면으로 들어갑니다.
3. 상단 탭에서 **[Volumes]** 탭을 클릭합니다.
4. **[New Volume]** 버튼을 클릭하여 볼륨을 추가합니다.
5. 마운트 경로(Mount Path)에 정확하게 **`/app/data`** 라고 입력합니다.
6. 오른쪽의 Save / Add 버튼을 눌러 저장합니다.
7. 볼륨을 추가하고 나면, 새로운 설정을 적용하기 위해 서비스가 잠시 재시작됩니다.

## ⚙️ 4단계: 포트(Port) 설정 및 도메인 발급

이제 앱이 Railway에서 켜진 상태이므로, 인터넷에서 누구나 접속할 수 있도록 주소(도메인)를 발급받아야 합니다.

1. 방금 전 열었던 서비스 설정 화면에서, 이번엔 상단의 **[Settings]** 탭을 클릭합니다.
2. 스크롤을 내려 **[Networking]** 섹션을 찾습니다.
3. **Public Networking** 영역에 있는 **[Generate Domain]** 버튼을 클릭합니다.
   - 클릭하시면 `(무작위이름).up.railway.app` 이라는 인터넷 주소가 생성됩니다. (이것이 홈페이지 주소입니다!)
   - *팁: 나중에 직접 구매한 도메인(예: .com, .kr)을 연결하시려면 Custom Domain 메뉴를 사용하시면 됩니다.*

---

## 💡 5단계: 관리자 비밀번호 등 환경변수 설정 (옵션)

개발할 때 쓰던 `.env` 설정(`VITE_ADMIN_PASSWORD` 등)을 Railway 서버에도 알려주어야 합니다.

1. 상단의 **[Variables]** 탭을 클릭합니다.
2. **[New Variable]** 버튼을 눌러 다음 내용들을 추가합니다:
   - Variable Name: `VITE_ADMIN_PASSWORD`
   - Value: `원하는_관리자_비밀번호` (안 적으셔도 기본값 '큰문admin2024'가 사용됩니다.)

---

## ✅ 6단계: 브라우저에서 최종 확인

1. **[Settings]** 탭에서 생성했던 도메인 주소(예: `https://xxx.up.railway.app`) 버튼을 클릭해 새 창으로 엽니다.
2. 정상적으로 홈페이지가 열리는지 확인합니다.
3. 브라우저 주소창 뒤에 `/#/admin` 을 붙여 관리자 페이지로 접속해봅니다. (예: `https://xxx.up.railway.app/#/admin`)
4. 커뮤니티 글을 하나 작성해보고, 이미지를 첨부해봅니다. 정상적으로 저장되고 목록에 뜬다면 성공입니다!
5. *테스트 팁:* 글 작성 직후, Railway 변수(Variables) 탭에서 아무 설정이나 빈 값을 추가했다 지워보세요 (서버 재배포 트리거). 서버가 잠시 꺼졌다가 켜진 직후 다시 사이트를 들어갔을 때, 내가 작성한 글이 살아있다면 볼륨 연동이 완벽하게 된 것입니다.

> 🎉 배포가 끝났습니다! 이제 앞으로 수정사항이 생기면 언제든지 **`git commit & push`** 만 하시면, Railway가 알아서 변경점을 파악해 수 분 만에 자동 배포(Auto Deploy)를 진행합니다.
